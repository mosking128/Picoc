# PicoC 上位机工具

用于当前 `STM32H750 + PicoC + USART1` 固件的 Windows 串口上位机。

## 功能

- 串口连接、断开与心跳检测
- 实时控制台输出
- 手工发送 PicoC 命令
- 上传本地 `.c` 文件并执行
- 文件列表管理
- `执行选中` 与 `全部执行` 两种执行方式
- 批量测试结果汇总
- 调试工具栏：断点、继续、单步、表达式求值
- 保存控制台日志
- 显示 PicoC 当前模式：`REPL`、`LOAD`、`BUSY`

## 运行环境

- Windows
- Python 3.9+

## 安装依赖

```powershell
pip install -r requirements.txt
```

如果安装失败，先清掉代理环境变量再试：

```powershell
set HTTP_PROXY=
set HTTPS_PROXY=
pip install -r requirements.txt
```

## 直接运行

```powershell
python app.py
```

或直接双击：

```text
run_host.bat
```

## 打包 EXE

```powershell
build_exe.bat
```

打包完成后会生成两个目录：

```text
dist\PicoCHost
release\PicoCHost
```

给别人使用时，直接把 `release\PicoCHost` 整个文件夹发给对方即可。

## 基本使用流程

1. 给板子上电并连接串口。
2. 点击“刷新”，选择正确的 COM 口，波特率保持 `115200`。
3. 点击“连接”。
4. 等待板端输出 `picoc> ` 或 `load> `。
5. 添加一个或多个 `.c` 文件到列表。
6. 单文件执行：
   点击“执行选中”，执行完成后自动跳到列表中的下一个文件。
7. 批量执行：
   点击“全部执行”，工具会依次上传并汇总结果。
8. 如需中止当前上传，点击“中止”。
9. 如需保存日志，点击“保存日志”。

## 调试使用

### 断点

1. 在“行号”里输入要停下来的源码行号。
2. 点击“设断点”。
3. 上传并执行文件。
4. 命中后状态栏会显示当前行，上位机启用“继续”“单步”“求值”。

### 单步和继续

- 点击“单步”：执行下一条语句后再次停下。
- 点击“继续”：继续执行到下一个断点或文件结束。

### 表达式求值

- 在“表达式”输入框中输入 C 表达式，比如 `i`、`sum + 1`。
- 点击“求值”。
- 当前实现更适合整数类表达式，结果会通过控制台输出。

### 当前边界

- 断点当前针对上传执行的源码，文件名固定按 `serial_load` 处理。
- 行号按上传文件内容计算。
- 每次文件执行结束后，板端会自动清空本轮断点。

## 协议假设

- REPL 提示符：`picoc> `
- 文件模式提示符：`load> `
- 文件模式命令：`:load`、`:end`、`:abort`
- 文件执行完成后板端输出：`ready for next file`
- 调试命令：
  - `:bkpt <file> <line>`
  - `:bkptclear <file> <line>`
  - `:cont`
  - `:step`
  - `:eval <expr>`

## 说明

- 当前工具只适配这套 STM32 PicoC 固件。
- 文件上传使用纯文本方式发送，只统一换行格式，不改写源码内容。
- 不包含源码编辑器和二进制下载协议。
