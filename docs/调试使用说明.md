# PicoC 断点调试与单步执行说明

## 1. 当前支持范围

本工程当前已经支持一套可用的 PicoC 串口调试流程，能力包括：

- 断点设置
- 断点清除
- 单步执行
- 继续执行
- 断点停下后做表达式求值
- 查看当前作用域变量
- 断点停下后修改变量值
- Watch 监视关键变量变化

这套能力当前主要面向“整文件上传执行”场景，不是桌面 IDE 那种完整源码级调试器。

## 2. 调试对象是什么

当前调试对象是通过 `:load` 上传并执行的源码。

板端执行整文件时，会创建一个隔离的 PicoC 实例来跑上传内容，同时把主实例上的断点复制进去。因此：

- 断点对上传文件生效
- 断点不会长期残留到下一次文件执行
- 文件执行结束后，当前断点会被清空

当前断点文件名按 `serial_load` 处理。

这意味着：

- 上位机设断点时，文件名固定使用 `serial_load`
- 实际需要你输入的是“上传文件中的行号”

## 3. 支持的调试命令

### 板端接收

- `:bkpt <file> <line>`
- `:bkptclear <file> <line>`
- `:cont`
- `:step`
- `:eval <expr>`
- `:vars`
- `:set <name> <value>`
- `:abort`

### 板端回传

- `:break <file> <line> <col>`
- `:step <file> <line> <col>`
- `:ok bkpt`
- `:ok bkptclear`
- `:ok eval`
- `:var <type> <name> <value>`
- `:ok vars`
- `:ok set`
- `:err ...`

## 4. 上位机使用流程

### 4.1 设置断点

1. 连接串口并等待 `picoc> `。
2. 在调试工具栏“行号”输入框中填写源码行号。
3. 点击“设断点”。
4. 上位机会发送：

```text
:bkpt serial_load <line>
```

### 4.2 执行文件

1. 选择 `.c` 文件。
2. 点击“执行选中”或“全部执行”。
3. 文件上传完成后开始执行。
4. 当执行到断点位置时，板端输出：

```text
:break serial_load <line> <col>
```

上位机会把它转成更友好的提示，并启用：

- `继续`
- `单步`
- `求值`
- `变量监视`
- `Watch`

### 4.3 单步执行

点击“单步”后，上位机发送：

```text
:step
```

板端继续执行一条语句，然后再次在下一处停下，并输出：

```text
:step serial_load <line> <col>
```

### 4.4 继续执行

点击“继续”后，上位机发送：

```text
:cont
```

程序会继续跑，直到：

- 命中下一个断点
- 文件执行结束
- 脚本报错
- 用户中止

### 4.5 表达式求值

点击“求值”后，上位机会发送：

```text
:eval <expr>
```

当前实现会把表达式包装成：

```c
printf("%d\n", (<expr>));
```

所以当前更适合：

- 整数变量
- 整数表达式
- 返回值可按 `%d` 打印的结果

如果表达式本身出错，板端会回：

- `:err eval ...`
- 或 `:err eval failed`

### 4.6 变量查看

断点命中或单步停下后，上位机会自动发送：

```text
:vars
```

板端会逐个回传当前可见变量，例如：

```text
:var i counter 12
:var p ptr 0x24001234
:ok vars
```

上位机会把这些结果填充到“变量监视”表格中，方便直接查看当前作用域变量。

### 4.7 修改变量值

在“变量监视”表格中双击某个变量的“值”单元格并修改后，上位机会发送：

```text
:set <name> <value>
```

如果修改成功，板端回：

```text
:ok set
```

如果修改失败，板端会回：

```text
:err set ...
```

当前这套修改能力更适合基础整数、浮点、字符这类能直接解析的标量变量。

### 4.8 Watch 监视

1. 在“变量监视”中选中变量。
2. 点击“加监视”把变量加入 `Watch` 表格。
3. 后续每次断点命中或单步停下后，`Watch` 会自动刷新。
4. 如果变量值发生变化，上位机会以 `旧值 → 新值` 的形式高亮显示。

## 5. 当前限制

- 当前断点主要面向上传文件，不是 REPL 历史输入。
- 行号必须和上传文件文本对应。
- 断点按 `serial_load` 这个逻辑文件名匹配。
- `:eval` 当前不是通用类型打印器，更偏向整数表达式求值。
- `:set` 当前主要面向基础标量变量，不覆盖复杂结构体成员的通用编辑。
- Watch 在停点时刷新，不是目标运行期间的实时采样。
- 每次整文件执行结束后，断点会自动清空，需要下次重新设置。
