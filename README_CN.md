# PicoC on STM32H750

[English](README.md) | [中文](README_CN.md)

本项目将 PicoC 移植到 STM32H750，在 `USART1` 上提供交互式 REPL、整文件上传执行和 Windows 上位机工具，目标是用尽量小的代价跑通 MCU 侧 C 脚本解释工作流，并沉淀一套可复用的 PicoC 嵌入式移植思路。

## 功能特性

- 基于 `USART1` 的 PicoC 交互式 REPL
- 基于 `:load / :end / :abort` 的整文件上传执行
- 支持多行源码输入和整文件运行
- 支持基础交互式调试：断点、单步、表达式求值、变量修改与 Watch 监视
- 提供 STM32H750 的 Keil MDK 工程
- 提供 Windows 上位机工具，用于串口控制、文件上传、批量测试和调试

## 调试能力

当前上位机已经支持一套可用的调试流程：

- 设置断点
- 清除断点
- 单步执行
- 继续执行
- 停在断点后做表达式求值
- 查看当前作用域变量
- 断点停下后直接修改变量值
- 将关键变量加入 Watch 持续观察变化

### 调试工具栏

上位机窗口底部常驻调试工具栏：

| 区域 | 控件 | 说明 |
|------|------|------|
| 断点 | `行号` + `设断点` / `清断点` | 在上传文件执行前设置或清除断点 |
| 执行 | `继续` / `单步` | 继续运行或按语句单步 |
| 表达式 | `表达式` + `求值` | 在当前断点上下文里求值 |
| 变量监视 | `变量监视` 表格 | 查看当前可见变量，并可双击数值直接修改 |
| Watch | `Watch` 表格 + `加监视` / `移除选中` | 跟踪关键变量在多次停点间的变化 |

### 调试流程

1. 连接板卡，等待出现 `picoc> `。
2. 输入断点行号，例如 `5`，点击“设断点”。
3. 上传并执行 `.c` 文件。
4. 命中断点后，上位机会显示 `已中断: 第5行`。
5. 程序停下后，“变量监视”会刷新当前作用域中的可见变量。
6. 可双击变量值直接修改，也可把选中的变量加入 `Watch` 持续观察。
7. 点击“单步”可继续逐语句执行，或输入 `x + 1` 这类表达式后点击“求值”。
8. 点击“继续”可继续运行到下一个断点或程序结束。
9. 每次文件执行结束后，当前断点会自动清空。

### 当前边界

- 调试当前主要针对上传执行的 `serial_load` 源码。
- 断点行号按上传文件内容计算，不针对 REPL 历史输入。
- `:eval` 当前内部会包装成 `printf("%d\\n", (...))`，因此更适合整数类表达式。
- 断点会复制到隔离的文件执行实例中，运行结束后自动清理。
- 变量修改当前主要覆盖调试输出里能直接表示的基础标量类型。
- Watch 值会在每次断点/单步停下时刷新，不是运行过程中的连续采样。

### 调试协议扩展

主机与板端之间的调试命令当前包括：

| 方向 | 命令 | 说明 |
|------|------|------|
| 主机 → 板端 | `:bkpt <file> <line>` | 设置断点 |
| 主机 → 板端 | `:bkptclear <file> <line>` | 清除断点 |
| 主机 → 板端 | `:cont` | 继续执行 |
| 主机 → 板端 | `:step` | 单步执行 |
| 主机 → 板端 | `:eval <expr>` | 表达式求值 |
| 主机 → 板端 | `:vars` | 枚举当前可见变量 |
| 主机 → 板端 | `:set <name> <value>` | 修改变量值 |
| 板端 → 主机 | `:break <file> <line> <col>` | 断点命中通知 |
| 板端 → 主机 | `:step <file> <line> <col>` | 单步停点通知 |
| 板端 → 主机 | `:var <type> <name> <value>` | 单个变量信息 |
| 板端 → 主机 | `:ok vars` | 变量枚举结束 |
| 板端 → 主机 | `:ok set` / `:err set ...` | 变量修改结果 |

## 固件快速开始

1. 用 Keil 打开 [MDK-ARM/UART_DMA_H750.uvprojx](MDK-ARM/UART_DMA_H750.uvprojx)。
2. 编译并烧录到 STM32H750。
3. 打开 `USART1`，波特率 `115200`。
4. 上电后等待 `picoc> ` 提示符。

## 上位机快速开始

源码方式运行：

```powershell
cd tools/picoc_host
run_host.bat
```

打包方式运行：

- 运行 `build_exe.bat`
- 将 `tools/picoc_host/release/PicoCHost` 整个目录发给其他用户

## 典型使用流程

### 文件上传执行

1. 连接板卡并打开上位机。
2. 等待 `picoc> `。
3. 向文件列表中添加 `.c` 文件，选中后点击“执行选中”。
4. 上位机会自动上传源码，目标端执行后在控制台输出结果。

### 断点调试

1. 连接设备，输入断点行号并点击“设断点”。
2. 上传执行文件。
3. 程序停下后，通过“变量监视”查看变量，必要时直接改值并加入 `Watch`。
4. 结合“单步”“求值”“继续”检查和控制运行状态。
5. 该轮执行结束后，断点自动清空。
